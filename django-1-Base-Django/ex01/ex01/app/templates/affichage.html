Processus d'affichage d'une page statique

Le processus pour afficher une page statique avec Django suit les étapes suivantes :

  1. Requête HTTP: L'utilisateur envoie une requête HTTP en accédant à une URL spécifique.
  2. URLconf: Django correspond l'URL demandée avec une vue définie dans le fichier urls.py.
  3. Vue: La vue correspondante est appelée. Elle peut être une fonction ou une classe.
  4. Rendu du template: La vue utilise un template (HTML) pour générer la réponse HTTP.
  5. Réponse HTTP: Django renvoie une réponse HTTP contenant le contenu généré.

Ce processus permet à Django de séparer les responsabilités (logique, présentation, routage) pour faciliter le développement.
