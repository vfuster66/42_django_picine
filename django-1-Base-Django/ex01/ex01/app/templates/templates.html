Le moteur de template Django

Le moteur de template de Django permet de séparer la logique métier et l'affichage. Voici quelques-unes de ses fonctionnalités principales :

Blocs

Les blocs sont définis dans le template parent avec la syntaxe {% verbatim %}{% block nom_du_bloc %}{% endblock %}{% endverbatim %}. Les templates enfants peuvent remplacer ou compléter ces blocs. 


{% verbatim %}
{% block content %}
    
Contenu spécifique à une page

{% endblock %}
{% endverbatim %}

Boucles for

Permet d'itérer sur des listes ou des objets passés dans le contexte : 


{% verbatim %}
{% for item in items %}
    
  • {{ item }}
    • 
{% endfor %}
{% endverbatim %}

    Structures de contrôle if

    Les conditions permettent de modifier le rendu selon le contexte : 

    
{% verbatim %}
{% if user.is_authenticated %}
    
    Bonjour {{ user.username }} !
    
{% else %}
    
    Bienvenue, visiteur anonyme !
    
{% endif %}
{% endverbatim %}

    Affichage des variables

    Les variables passées par le contexte peuvent être affichées avec la syntaxe {{ variable }}. 

    
{% verbatim %}

    Le titre est : {{ title }}
    
{% endverbatim %}

    Django s'assure que les données affichées sont sécurisées, notamment en échappant les caractères HTML.
